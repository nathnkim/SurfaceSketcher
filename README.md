# Surface Sketcher

A lightweight, pen-first sketching app for Windows 11 / Surface devices, built with Electron.

## Features

- **Surface Pen drawing** — pressure-sensitive, variable-width strokes via the Pointer Events API (`pointerType: 'pen'`, `e.pressure`), reading `getCoalescedEvents()` so fast strokes capture the pen's full hardware sample rate instead of one point per animation frame.
- **Flip to erase** — flipping the pen to its eraser end (detected via the `buttons` eraser bit / `button === 5`) erases whole strokes under contact.
- **Pen barrel (side) button → screenshot** — captures the current drawing's bounding box as a transparent PNG. Saved silently into a `Screenshots/` folder inside your open workspace (or via a save dialog if no workspace is open). This is triggered only by the pen's side button — see "Known limitations" below for why there's no equivalent toolbar button or top-button binding.
- **5-swatch color palette** — click a swatch to select it, double-click to recolor it (native color picker).
- **Sidebar file tree** — open a workspace folder, create/rename/delete `.skx` sketch files and subfolders anywhere in it. Click a folder to target it for the next "New File"/"New Folder" (shown with a dashed outline); click a file to target its parent folder; click empty space below the tree to retarget the workspace root. Deletes are soft (moved to a `.trash` folder, not permanently removed).
- **PNG / SVG export** — both crop to the bounding box of your drawing plus a margin you control in Settings. PNG export has a transparent background. SVG export converts each stroke into the same filled variable-width vector outline used for on-screen rendering, so the two are visually identical.
- **Pinch to zoom, two-finger rotate/pan** — standard touch gesture math on the canvas. Mouse/trackpad fallback: Ctrl+wheel to zoom, wheel to pan, spacebar+drag to pan.
- **Single pen tool** — no tool switching; color + eraser-by-flip is the whole interaction model, as requested.
- **Toggleable dot grid** — visual drawing aid only, rendered under strokes, excluded from all exports.
- **Infinite canvas** — opens directly into an unbounded drawing surface (OneNote-style); pan freely in any direction, no page boundaries.
- **Autosave** — edits save to the open `.skx` file automatically (debounced ~400ms after you stop drawing). An amber dot next to the file name shows when there are unsaved changes.
- **Undo/redo** — Ctrl+Z / Ctrl+Y, plus toolbar buttons.

## Requirements

- **Windows 10 or 11.** The app is Windows-only by design (Surface Pen input handling, NSIS packaging) — no macOS/Linux build target is configured.
- **Node.js 18+** to install dependencies and run from source.
- No Visual Studio or native build tools needed — Electron ships prebuilt binaries, and this project has no native (node-gyp) dependencies.
- A Surface Pen (or other pointer-events-capable pen digitizer) for the pressure/eraser/barrel-button features; mouse/trackpad works for drawing, panning, and zooming but has no pressure or eraser-flip equivalent.

Tested on Windows 11 with a Surface Pen — pressure-sensitive drawing, flip-to-erase, and the barrel-button screenshot have all been confirmed working on real hardware.

## Install & run from source

```
git clone https://github.com/nathnkim/SurfaceSketcher.git
cd SurfaceSketcher
npm install
npm start
```

`npm install` pulls only two dev dependencies (`electron`, `electron-builder`) — no native modules, so this should complete without a compiler toolchain on a stock Windows machine.

## Building a distributable installer

```
npm run dist
```

This runs `electron-builder` and produces an NSIS installer in `release/`, configured to install with Start Menu and Desktop shortcuts (so the app shows up in Windows Search once installed). The packed app should be well under 200MB installed (Electron's baseline).

**Known build-machine caveat:** `electron-builder` downloads a macOS code-signing helper (`winCodeSign`) as part of its packaging pipeline even for a Windows-only build, and extracting it requires the "create symbolic link" privilege. Standard (non-elevated, non-Developer-Mode) Windows accounts don't have this privilege, so `npm run dist` can fail with a `Cannot create symbolic link` error during that extraction step. Two ways around it, neither of which this project changes for you automatically:

- Enable **Developer Mode**: Settings → Privacy & security → For developers → Developer Mode, or
- Run `npm run dist` from an **Administrator** terminal.

If you just want a runnable shortcut without dealing with the installer, `npm start` plus a Desktop/Start Menu `.lnk` pointing at `node_modules\electron\dist\electron.exe "<path to this folder>"` (working directory set to the same folder) achieves the same "launch from Windows Search or Desktop" result without needing the installer at all.

## Custom app icon

Drop a Windows icon file at `build/icon.ico` (multi-resolution `.ico`, ideally containing 16/32/48/256px sizes). It's already wired up in two places:

- `package.json`'s `build.win.icon`, used by the packaged installer/exe, and
- `main/main.js`'s `BrowserWindow` options, used for the taskbar/title-bar icon during `npm start`.

No icon is committed to this repo (see `build/README.md`) — until you add one, the app falls back to Electron's default icon.

## Architecture

```
main/           Electron main process
  main.js         window creation, settings persistence (userData/settings.json),
                  workspace/file-tree filesystem IPC, export & screenshot IPC
  preload.js      contextBridge-exposed API surface (window.api.*) — the renderer
                  never gets direct Node/fs access (contextIsolation: true,
                  nodeIntegration: false)

renderer/       UI, all pure DOM/Canvas — no Electron/Node APIs outside app.js's
                use of window.api
  engine.js       SketchEngine: infinite-canvas rendering, pointer input
                  (pressure, flip-to-erase, coalesced-event sampling),
                  pinch/rotate/pan gesture math, undo/redo, Catmull-Rom stroke
                  smoothing, and the shared stroke-outline geometry used by
                  both canvas rendering and SVG export
  filetree.js     sidebar workspace tree: list/create/rename/delete, and the
                  "selected directory" targeting for New File/Folder
  export.js       PNG/SVG export (bounding-box + margin crop)
  dialog.js       in-page replacement for window.prompt(), which Electron does
                  not implement (throws "prompt() is not supported")
  app.js          wires engine + file tree + palette + top bar + settings
                  panel together; the only renderer file that touches DOM
                  chrome outside the canvas

build/          Local-only assets (app icon) — see build/README.md
```

## The `.skx` file format

Each sketch is a small JSON file: `{ version, strokes: [{ id, color, baseWidth, points: [{x, y, pressure}] }], view }`. It's the save/autosave format; PNG and SVG are export-only and don't round-trip back into `.skx`.

## Known limitations

- **The Surface Pen's *top* button** is intercepted by Windows itself (Snip & Sketch, OneNote, etc.) before any button-press event reaches a regular desktop app — Electron included. There's no supported way for a non-UWP Win32/Electron app to capture that event directly. To route the top button to this app's screenshot feature instead, remap it in **Windows Settings → Bluetooth & devices → Pen & Windows Ink** — a system-wide setting the user configures, not something the app can do for itself.
- **Building the NSIS installer** requires Developer Mode or an elevated terminal on the build machine — see "Building a distributable installer" above.
